# System Module Tree Analysis

- **Phase 1: Staff & Administrative Operations** (Staff, Librarian, Admin)
    - **Inventory Management** (`/dashboard`)
        - Browse library stock and inventory
        - Category-based filtering and sorting
    - **Catalog Management** (`/catalog`)
        - Resource addition interface (`/catalog/add`)
    - **Circulation Desk** (`/circulation`)
        - **Checkout Flow**: Student QR identification → Resource QR verification
        - **Return Flow**: Resource QR scanning → Borrow validation
    - **User Directory** (`/users`)
        - Profile browsing and member auditing
    - **Borrow History** (`/history`)
        - Global transaction logs and auditing
    - **Policies & Settings** (`/policies`)
        - Circulation & Reservation rules
        - Identity Card & Category configuration
        - Announcements & Support FAQs
    - **Audit Logs** (`/audit`) - *Admin Only*
        - Immutable tracking of sensitive system actions
    - **Security** (`/security`)
        - Session and access control management

- **Phase 2: Student Experience** (Students)
    - **Student Dashboard** (`/dashboard`)
        - Digital Library Card (QR identity)
        - Active borrow and reservation statistics
    - **Resource Catalog** (`/student-catalog`)
        - Book discovery and reservation requests
    - **My History** (`/history`)
        - Personal borrowing and return timeline
    - **Notifications** (`/notifications`)
        - Real-time alerts for overdue items and system updates
    - **Identity & Security** (`/security`)
        - Account security and credential management
